# zhibo-toolkit

直播平台视频下载工具集（叶叶财经小灶课等）。

## 工具

### download_course.sh

自动下载直播课程并重命名。

```bash
./download_course.sh <课程页面URL> [cookies文件] [输出目录]
```

**示例：**
```bash
# 基本用法（自动使用同目录下的cookies.txt和videos目录）
./download_course.sh "https://szb135927.livec.shangzhibo.tv/watch/11842299"

# 指定输出目录
./download_course.sh "https://xxx.livec.shangzhibo.tv/watch/12345" "" ./my_videos
```

**依赖：**
- yt-dlp: `pip install yt-dlp`
- curl
- ffprobe (可选)

---

### rename.ps1

按时间顺序对 `videos/` 目录下的视频进行批量重命名，**第N节** 号从前到后递增。

```powershell
# 1. 编辑 $Map 把当前的"原始文件名 -> 目标文件名"映射填好
# 2. 执行脚本
powershell -ExecutionPolicy Bypass -File .\rename.ps1
```

## 命名规则（必须遵守）

### 视频文件命名

所有下载到 `videos/` 目录的课程视频统一使用以下格式：

```
课程名_第NN节_<日期>_<标题>.mp4
```

- **`课程名`**：固定为 `叶叶财经小灶课`（或对应课程名）
- **`第NN节`**：两位数字（01, 02, 03, ...），从**第01节**起按**时间先后**递增
- **`<日期>`**：原始日期，例如 `5月13日`、`5月18日`、`6月1日`（**`5.18` 必须规范化为 `5月18日`**）
- **`<标题>`**：课程原始标题，保留原样（如 `叶叶铁粉小灶课`、`叶叶小灶课第5节`、`微信视频`）

### 命名示例

```
叶叶财经小灶课_第01节_5月13日叶叶铁粉小灶课.mp4
叶叶财经小灶课_第02节_5月14日叶叶财经小灶课第二节.mp4
叶叶财经小灶课_第03节_5月15日叶叶财经小灶课.mp4
叶叶财经小灶课_第04节_5月17日叶叶小灶课第4节.mp4
叶叶财经小灶课_第05节_5月18日叶叶小灶课第5节.mp4
...
叶叶财经小灶课_第16节_5月31日叶叶小灶课第16节.mp4
叶叶财经小灶课_第17节_6月1日微信视频.mp4
```

### 关键规则

1. **课次号连续**：从 `第01节` 开始严格连续递增，不跳号
2. **时间顺序**：课次号 = 课程日期从早到晚的顺序（第01节 = 最早的课）
3. **日期格式统一**：`5.18` → `5月18日`，确保所有日期都带 `月` `日`
4. **保留原标题**：标题中的"第N节"、"第二节"等保留，便于追溯
5. **特殊标题处理**：微信上传的非标准标题（如 `微信视频2026-06-01_191057_287.mp4`）简化为 `微信视频`

## 项目结构

```
zhibo-toolkit/
├── download_course.sh    # 下载脚本
├── rename.ps1            # 批量重命名脚本
├── cookies.txt           # cookies文件（不入库，需用户自备）
├── videos/               # 视频输出目录（不入库）
├── text/                 # 字幕/转写文本（不入库）
├── .playwright-cli/      # 浏览器缓存（不入库）
├── .gitignore
└── README.md
```

## 支持平台

- shangzhibo.tv (尚品直播)
- 其他使用 m3u8 HLS 流媒体的平台

## 使用流程

1. 从浏览器导出 cookies 为 Netscape 格式，保存为 `cookies.txt`
2. 运行下载脚本：
   ```bash
   ./download_course.sh "课程页面URL"
   ```
3. 视频会自动下载到 `videos/` 目录
4. 若是首次批量下载或新增了课程，运行 `rename.ps1` 按时间顺序整理文件名
